Opal is a complex and powerful software application for storing and integrating heterogeneous biobank data. We list here some of its key features.

Data Warehouse

Opal's database implements a generalized Entity-Attribute-Value (EAV) data model which enables to store values on an unlimited number of variables without any database structure alterations. Customized variable dictionaries are simply defined in Excel files that are then uploaded into Opal. Here are some of the main features and functionalities of the Opal's database.

  • Able to integrate multiple sources of data (datasource)
  • Variable dictionary manager
  • Import data in CSV, Excel and Onyx format
  • Connect to existing SQL databases
  • Uniform variable and data representation
  • Any number of variables and entities
  • Any type of "entity"
  • Statistical summary

Derive Variables

Most of the time, researchers need to create research datasets that contain new derived variables. For example, a researcher might be interested in the Body Mass Index (BMI) for each participant which is obtained by dividing the measured weight by the height. Opal provides all the infrastructure to derive new variables as summarized in this list of features.

  • Free-form algorithms
  • Measurement unit conversion
