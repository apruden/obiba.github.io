Data Warehouse

Opal's database structure enables to store values on an unlimited number of variables without any database structure alterations. Customized variable dictionaries are simply defined in Excel files that are then uploaded into Opal. Here are some of the main features and functionality of the Opal's database:

  • Use MongoDB or Mysql as database software backend.
  • Import data in CSV, Excel, SPSS and Onyx format.
  • Can connect with multiple data sources such as existing SQL databases and LimeSurvey software to import data directly to Opal.
  • Include an advanced variable dictionary manager.
  • Data organized in projects and tables.
  • Uniform variable and data representation.
  • Unlimited number of variables and records.
  • Can store data on any type of "entity", such as subject, sample, geographic area, etc.
  • Support any data type (e.g, decimal, binary) including geographical.
  • Advanced indexing functionality using ElasticSearch.

Views and Derived Variables

Most of the time, researchers need to create datasets that contain new derived variables. To this end, Opal provides all the infrastructure to create virtual tables or “views” of derived variables that can be persisted on disk or exported into files afterward. Derived variables are defined using JavaScript code executed on demand to create new datasets from raw data stored in Opal. Here are the main features of this infrastructure:

  • Advanced JavaScript library of util functions commonly used to derive new variable (e.g. unit conversion) See http://wiki.obiba.org/display/OPALDOC/Magma+Javascript+APIMagma Javascript API.
  • Graphical wizard to easily recode variables without writing Javascript code.
  • instant summary statistics computation of the new derived variables.

Privacy, Confidentiality and Security

Ensuring participant privacy and data confidentiality is a fundamental issues in any biobank or epidemiological study. Opal provides users with state-of-the-art infrastructure for managing data encryption, participant identifiers and data access privileges. Here is a summary of Opal's features regarding privacy, confidentiality and security:

  • Public Key Infrastructure (PKI) allowing Opal to create and use public-private key pairs for encrypting and decrypting data.
  • Authentication using either certificates and username/password mechanisms.
  • Advanced participant identifiers manager enabling multiple identifiers per participant.
  • Distinct and highly secure database for storing participant identifiers.
  • Granular permission management down to the variable level.
  • Restful services using HTTPS protocol.

Indexing

Opal will automatically index all the data that is added to it. This allows for very fast retrieval and complex querying of the data. Furthermore, the indexing can be managed by a cluster of machines for better performance, thanks to the ElasticSearch software:

  • Clustered indexing
  • Real-time search capability

Web Services (API)

Opal is built on RESTful services. Everything in Opal is accessible through a URL. The resources can be obtained in JSON or binary form (Protobuf). Any client that can make an HTTPs request can be a client to an Opal server.
